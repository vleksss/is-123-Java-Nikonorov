# Auction Web Application

Курсовой проект по теме **«Веб-приложение аукцион»**.

Приложение реализовано на **Java 17** с использованием **Spring Boot**, **Spring Security**, **Spring Data JPA** и **RED Database**.  
Интерфейс проекта доступен в трёх вариантах: **Thymeleaf**, **Mustache** и **FreeMarker**.

---

## Функциональность

- регистрация и авторизация пользователей
- ролевая модель пользователей
- создание аукционов владельцем
- просмотр списка аукционов
- размещение ставок
- хранение пользователей, аукционов и ставок в RED Database
- три разных веб-движка интерфейса
- unit-тесты основной бизнес-логики
- GitHub Actions для автоматического запуска тестов

## Основные элементы системы

### Три веб-движка интерфейса

В проекте реализованы три разных серверных шаблонизатора:

- **Thymeleaf**  
  Основной шаблонизатор Spring Boot. Используется для отображения страниц с помощью атрибутов `th:*`.

- **Mustache**  
  Шаблонизатор с простым синтаксисом `{{...}}`.

- **FreeMarker**  
  Шаблонизатор с синтаксисом `${...}` и директивами `<#...>`.

Все три движка работают с одной и той же логикой и одной базой данных.

Адреса страниц:

- `http://localhost:8080/thymeleaf/auctions`
- `http://localhost:8080/mustache/auctions`
- `http://localhost:8080/freemarker/auctions`

---

### Три роли в системе

В приложении используются три роли:

- **USER**  
  Может просматривать аукционы и делать ставки.

- **OWNER**  
  Может создавать аукционы, просматривать их и размещать ставки.

- **ADMIN**  
  Имеет доступ к административной части и управлению пользователями.

---

### Три статуса аукциона

Каждый аукцион может находиться в одном из трёх состояний:

- **DRAFT**  
  Аукцион создан, но ещё не начался.

- **ACTIVE**  
  Аукцион активен, ставки доступны.

- **CLOSED**  
  Аукцион завершён, новые ставки больше не принимаются.

## Структура базы данных

В проекте используются таблицы:

### `APP_USER`
Содержит пользователей системы:
- логин
- email
- пароль
- роль
- признак активности

### `AUCTION`
Содержит аукционы:
- название
- описание
- начальная цена
- дата начала
- дата окончания
- статус
- владелец

### `BID`
Содержит ставки:
- сумма ставки
- дата и время
- пользователь
- аукцион

## Подготовка базы данных


SQL-скрипты находятся в папке:

- `src/main/resources/db/schema-red.sql`
- `src/main/resources/db/demo-data.sql`

### Порядок подготовки базы

1. Создать пустую базу RED Database.
2. Выполнить `schema-red.sql`.
3. Выполнить `demo-data.sql`.

---

## Параметры запуска

Подключение к базе данных задаётся через переменные окружения:

- `DB_HOST`
- `DB_PORT`
- `DB_PATH`
- `DB_USERNAME`
- `DB_PASSWORD`

Значения по умолчанию:

- `DB_HOST=localhost`
- `DB_PORT=3050`
- `DB_PATH=C:/basa/auction.fdb`
- `DB_USERNAME=SYSDBA`
- `DB_PASSWORD=masterkey`

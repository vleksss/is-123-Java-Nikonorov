# Auction Web Application

Курсовой проект по теме **«Веб-приложение аукцион»**.

Приложение реализовано на **Java 17** с использованием **Spring Boot**, **Spring Security**, **Spring Data JPA** и **RED Database / Firebird**.

Интерфейс проекта доступен в трёх вариантах:

- **Thymeleaf**
- **Mustache**
- **FreeMarker**

Все три веб-движка работают с одной бизнес-логикой и одной базой данных.

---

## Функциональность

- регистрация и авторизация пользователей
- ролевая модель пользователей
- создание аукционов владельцем
- просмотр списка аукционов
- размещение ставок
- просмотр личного кабинета пользователя
- просмотр кабинета владельца аукционов
- административная часть
- хранение пользователей, аукционов и ставок в RED Database / Firebird
- автоматическое создание базы данных при запуске приложения
- три разных веб-движка интерфейса
- unit-тесты основной бизнес-логики
- GitHub Actions для автоматического запуска тестов

## Основные элементы системы

### Три веб-движка интерфейса

В проекте реализованы три серверных шаблонизатора.

**Thymeleaf**  
Основной шаблонизатор Spring Boot. Используется для отображения страниц с помощью атрибутов `th:*`.

**Mustache**  
Шаблонизатор с простым синтаксисом `{{...}}`.

**FreeMarker**  
Шаблонизатор с синтаксисом `${...}` и директивами `<#...>`.

Все три движка работают с одной и той же логикой приложения и одной базой данных.

Основные страницы:

- `http://localhost:8080/thymeleaf/auctions`
- `http://localhost:8080/mustache/auctions`
- `http://localhost:8080/freemarker/auctions`

---

### Роли пользователей

В приложении используются три роли.

**USER**  
Обычный пользователь. Может просматривать аукционы и делать ставки.

**OWNER**  
Владелец аукционов. Может создавать аукционы, просматривать свои аукционы и также размещать ставки.

**ADMIN**  
Администратор. Имеет доступ к административной части и управлению пользователями.

---

### Статусы аукциона

Каждый аукцион может находиться в одном из трёх состояний.

**DRAFT**  
Аукцион создан, но ещё не начался.

**ACTIVE**  
Аукцион активен, ставки доступны.

**CLOSED**  
Аукцион завершён, новые ставки больше не принимаются.

Статусы аукционов обновляются при обращении к данным приложения. Если время начала или окончания аукциона уже наступило, статус автоматически приводится к актуальному состоянию.

## Структура базы данных

В проекте используются три основные таблицы.

### `APP_USER`

Содержит пользователей системы:

- логин
- email
- пароль
- роль
- признак активности

### `AUCTION`

Содержит аукционы:

- название
- описание
- начальная цена
- дата начала
- дата окончания
- статус
- владелец

### `BID`

Содержит ставки:

- сумма ставки
- дата и время
- пользователь

## База данных

В проекте используется RED Database / Firebird.

База данных создаётся автоматически при запуске приложения. Если файл базы данных отсутствует, приложение создаст его самостоятельно и подготовит необходимые таблицы.



По умолчанию база данных создаётся в папке проекта по пути:

```text
database/auction.fdb
```


---

## Изменение расположения базы данных

Если нужно хранить базу данных в другом месте, можно указать переменную окружения `DB_PATH`.



```cmd
set DB_PATH=D:/auction_db/auction.fdb
mvnw.cmd spring-boot:run
```

---

## Параметры запуска

Подключение к базе данных задаётся через переменные окружения.

Доступные параметры:

- `DB_HOST`
- `DB_PORT`
- `DB_PATH`
- `DB_USERNAME`
- `DB_PASSWORD`

Значения по умолчанию:

```text
DB_HOST=localhost
DB_PORT=3050
DB_PATH=<папка_проекта>/database/auction.fdb
DB_USERNAME=SYSDBA
DB_PASSWORD=masterkey
```

Если переменные окружения не заданы, приложение использует значения по умолчанию.
